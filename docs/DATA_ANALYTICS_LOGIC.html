Diploma Data Analysis Dashboard

Data Analytics Logic — Presentation document

1. Overview

The application accepts three Excel result files (CO1K, CO3K, CO5K), parses multi-row headers and student marks, reads the AJ column (result column) for pass/fail and ATKT, computes subject averages and top 10, and visualizes results with charts and AI-generated insights. No database is used; data is held in React state.

2. Data Flow

Excel (CO1K, CO3K, CO5K) → Parse (xlsx → JSON with resultStatus from AJ column) → Analyze (pass/fail from AJ, subject averages, top 10, ATKT, insights) → Dashboard (cards, pie, line, per-file bar charts)
StageDescription
UploadUser selects Excel files; client sends to /api/analyze.
ParseServer uses xlsx to detect headers, extract student rows, subject marks, and the result column (AJ); returns JSON including resultStatus.
AnalyzeClient classifies pass/fail and ATKT from resultStatus; computes subject averages, top 10, and insights.
DisplayOverview cards, pie chart, line chart, per-file bar charts, and AI insights.

3. Excel Parsing Logic

Expected structure

  • Abbreviation row (e.g. row 4): Subject codes (OSY, STE, …) repeated per sub-column.
  • Passing Head row (e.g. row 5): SA-TH, FA-TH, TOTAL, SLA, SA-PR, FA-PR, TOTAL, …
  • Header row (e.g. row 6): Seat Number, Enrollment Number, Student Name, then max marks.
  • Data rows: One row per student.

Key rules

  • Header row is detected by searching for "seat", "enrollment", or "student name".
  • Columns are grouped by subject code from the abbreviation row; only columns with passing-head "TOTAL" are used per subject (avoids double-counting).
  • Result column (AJ): Detected by header keywords ("result", "class", "grade", "division", "status", "outcome") or column index 35 (Excel column AJ). Value stored as resultStatus (e.g. "First Class", "Fail", "A.T.K.T.").
  • Grand total column is detected for overall student total; values rounded to 2 decimal places.

4. Analysis Logic (Per Semester)

MetricLogic
Pass / FailFrom AJ column (resultStatus): "Fail" → fail; "A.T.K.T."/ATKT → ATKT (counted as failed); any other value (e.g. First Class, First Class with Distinction, First Class Conditional) → pass.
Subject averagesAverage marks per subject across students. "TOTAL" excluded from subject-wise bar charts.
Top 10Students sorted by total marks; top 10 shown with rank, name, total.
ATKTStudents whose AJ column value is "A.T.K.T." or "ATKT". Counted as failed; listed in ATKT section per semester.

5. AI Insights

Rule-based insights: pass rate comparison between semesters, ATKT count per semester (from result column), enrollment comparison, and best subject (highest average) per semester.

6. Dashboard Visualizations

Overview cardsTotal students, Pass %, Fail % (aggregated).
Pass vs Fail (pie)Proportion passed vs failed.
Line chartPass % (left Y-axis, 0–100) and Total students (right Y-axis) across CO1K, CO3K, CO5K.
Per-file bar chartSubject-wise average marks for that file only (e.g. "CO1K — Subject-wise average marks").
Top 10 / ATKTPer-semester tables; ATKT from result column.

7. Summary

The pipeline parses Excel with correct header and TOTAL-only subject columns, reads the AJ column for pass/fail and ATKT (no percentage logic), computes subject averages and top 10, aggregates for overview and pie chart, generates comparative insights, and renders one subject-wise bar chart per file plus line chart and tables.
